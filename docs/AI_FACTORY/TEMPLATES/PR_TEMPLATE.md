# PR Template

Use this checklist in future PR descriptions for this repository.

- [ ] Task scope is small and coherent
- [ ] Relevant code/docs were inspected before changes
- [ ] Existing validation commands were run before changes when applicable
- [ ] Targeted validation was run after changes
- [ ] `CURRENT_STATE.md` reviewed/updated if implementation changed
- [ ] `ACTIVE_WORK.md` reviewed/updated
- [ ] `HANDOFF.md` updated with latest result and next task
- [ ] Relevant subsystem/style/schema/QA docs updated
- [ ] No copyrighted style copying introduced
- [ ] Music/SFX priorities respected unless task was specifically about voice
- [ ] Downstream asset organization considered
