# Phase Plan

## Dependency order

1. documentation foundation
2. audio plan and request schemas
3. deterministic batch generation support
4. review and QA automation
5. `GameRewritten` import integration
6. backend quality upgrades
7. optional voice expansion

## Why this order

- without docs, future agents waste time rediscovering context
- without plan/request schemas, generation work stays ad hoc
- without deterministic provenance, asset libraries become hard to maintain
- without QA/review, larger generation volumes create noisy output
- backend work is more valuable after the workflow around it is stable
